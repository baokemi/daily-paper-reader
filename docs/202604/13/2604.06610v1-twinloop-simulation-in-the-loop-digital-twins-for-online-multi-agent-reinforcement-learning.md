## 摘要
去中心化的在线学习使得网络-物理多智能体系统能够在运行时进行自适应，但当操作环境发生变化时，已学习的策略往往需要经过大量反复试错才能恢复性能。为此，我们提出了 TwinLoop——一种面向在线多智能体强化学习的仿真闭环数字孪生框架。当上下文发生变化时，数字孪生被触发以重建当前系统状态，从最新的智能体策略进行初始化，并利用仿真中的“假设分析”加速策略改进，然后将更新后的参数同步回物理系统中的智能体。我们在一个包含变化工作负载和基础设施条件的车载边缘计算任务卸载场景中评估了 TwinLoop。结果表明，数字孪生可以提升环境变化后的自适应效率，并减少对昂贵在线试错过程的依赖。

## Abstract
Decentralised online learning enables runtime adaptation in cyber-physical multi-agent systems, but when operating conditions change, learned policies often require substantial trial-and-error interaction before recovering performance. To address this, we propose TwinLoop, a simulation-in-the-loop digital twin framework for online multi-agent reinforcement learning. When a context shift occurs, the digital twin is triggered to reconstruct the current system state, initialise from the latest agent policies, and perform accelerated policy improvement with simulation what-if analysis before synchronising updated parameters back to the agents in the physical system. We evaluate TwinLoop in a vehicular edge computing task-offloading scenario with changing workload and infrastructure conditions. The results suggest that digital twins can improve post-shift adaptation efficiency and reduce reliance on costly online trial-and-error.

## 一、研究背景与核心问题

### 1. 研究动机
- **问题**：在动态环境中运行的**多智能体系统**（如车载边缘计算场景）需要持续地自适应。但传统的**在线强化学习（RL）**在环境变化后往往依赖于昂贵的“试错”过程，导致性能恢复缓慢。
- **挑战**：
  - 环境变化引起策略失效，智能体需要重新探索并学习。

- **研究意义**：探索如何在环境变化时利用**数字孪生（Digital Twin, DT）**进行仿真加速，实现策略快速更新，以减轻实际环境中的探索成本。

## 二、方法论：TwinLoop 框架设计

### 1. 核心思想
TwinLoop 是一个**仿真闭环（simulation-in-the-loop）**数字孪生框架，用于加速去中心化多智能体强化学习的在线适应过程。

当检测到**环境上下文变化（context shift）**时：
1. 数字孪生（DT）被触发，重建系统当前状态；
2. 从物理系统中提取最新的智能体策略；
3. 在仿真环境中执行快速策略改进（what-if 分析）；
4. 最后将优化后的参数同步回物理系统。

该机制实现了“在虚拟环境中试错，在现实系统中部署”的安全适应思路。

### 2. 技术细节与算法流程

#### (1) 系统与强化学习模型
- 研究场景：**车载边缘计算（Vehicular Edge Computing, VEC）**任务卸载问题。
- 每辆车独立决策任务由本地计算还是卸载到路侧服务器（RSU）。
- 状态空间：包括车辆与服务器的处理率、队列长度、传输速率等。
- 动作空间：选择本地执行（`a_i=0`）或某一服务器（`a_i=j>0`）。
- 奖励函数：任务完成的负延迟值 `r_i = - t_e2e`，目标是最小化总延迟。

#### (2) 强化学习算法
- 使用 **Dueling Double DQN (D3QN)**：
  - 同时建模状态价值函数 `V(s)` 与动作优势函数 `A(s,a)`；
  - 双网络结构减少 Q 值过估；
  - 动作选择采用 **Boltzmann 策略**，以温度参数 τ 控制探索程度。

#### (3) TwinLoop 工作流程（算法 1）
分三阶段：
1. **Snapshot 阶段**：采集物理系统快照，包括服务器与车辆状态、任务速率及智能体权重。
2. **DT Training 阶段**：重建数字孪生，并以高探索率 τ₀ 进行加速策略更新。
3. **Weight Synchronisation 阶段**：完成训练后，更新权重 `Θ_PT ← Θ_DT` 同步回物理系统，恢复在线学习。

TwinLoop 的仿真训练速度约为物理系统的 **25×**。

## 六、主要结论与发现

1. TwinLoop 显著加快在环境变化后的策略收敛，减少平均和尾部延迟。
2. 在工作负载剧烈变化阶段（Phase 1、2），DT 助力效果最明显。
3. **DT 触发频率**需适度：
   - 过少触发 → 训练不足；
   - 过频触发 → 造成模型不稳定。
4. **训练时间**越长，策略适应性越好（T_DT=500 总体优于 T_DT=250）。
5. 多场景仿真在当前均质环境下收益有限，但在更复杂地理条件下可能更显著。
