## 摘要
流式视觉语言模型（VLMs）在接收指令提示和输入帧的在线流时，会持续生成响应。这是实现实时视觉助手的核心机制。现有的 VLM 框架主要在离线环境中评估模型。相比之下，流式 VLM 的性能取决于除了视频理解之外的其他指标，包括反应性（proactiveness），反映模型响应的及时性，以及一致性（consistency），衡量模型随时间推移响应的稳定性。为了解决这一限制，我们提出了 VSAS-Bench，这是一个用于视觉流助手的新框架和基准测试。与之前主要在视频输入上采用单轮问答的基准不同，VSAS-Bench 具有时间密集型标注，在不同输入领域和任务类型中包含超过 18,000 条标注。我们引入了标准化的同步和异步评估协议，以及能够分离并测量流式 VLM 不同能力的指标。利用该框架，我们对近期的视频和流式 VLM 进行了大规模评估，分析了在记忆缓冲区长度、内存访问策略和输入分辨率等关键设计因素下的准确率—延迟权衡，从而获得了若干实用洞见。最后，我们通过实证表明，传统的 VLM 可以在无需额外训练的情况下适应流式场景，并展示了这些改进模型优于近期的流式 VLM。例如，Qwen3-VL-4B 在异步协议下比我们基准中表现最好的流式 VLM Dispider 高出 3%。该基准和代码将发布在 https://github.com/apple/ml-vsas-bench。

## Abstract
Streaming vision-language models (VLMs) continuously generate responses given an instruction prompt and an online stream of input frames. This is a core mechanism for real-time visual assistants. Existing VLM frameworks predominantly assess models in offline settings. In contrast, the performance of a streaming VLM depends on additional metrics beyond pure video understanding, including proactiveness, which reflects the timeliness of the model's responses, and consistency, which captures the robustness of its responses over time. To address this limitation, we propose VSAS-Bench, a new framework and benchmark for Visual Streaming Assistants. In contrast to prior benchmarks that primarily employ single-turn question answering on video inputs, VSAS-Bench features temporally dense annotations with over 18,000 annotations across diverse input domains and task types. We introduce standardized synchronous and asynchronous evaluation protocols, along with metrics that isolate and measure distinct capabilities of streaming VLMs. Using this framework, we conduct large-scale evaluations of recent video and streaming VLMs, analyzing the accuracy-latency trade-off under key design factors such as memory buffer length, memory access policy, and input resolution, yielding several practical insights. Finally, we show empirically that conventional VLMs can be adapted to streaming settings without additional training, and demonstrate that these adapted models outperform recent streaming VLMs. For example, Qwen3-VL-4B surpasses Dispider, the best streaming VLM on our benchmark, by 3% under the asynchronous protocol. The benchmark and code will be available at https://github.com/apple/ml-vsas-bench.

## 一、研究背景与核心问题

- **研究动机**  
  随着大规模视觉语言模型（Vision-Language Models, VLMs）的发展，其在多模态理解和推理方面取得显著进展。然而，主流研究的评估方式仍集中在**离线视频理解**，例如单回合问答或静态视频分析，这与**实时流式视觉助手（Visual Streaming Assistant）**的实际使用场景存在巨大差距。  
  在真实应用中（如烹饪辅助、城市出行导航、为视障人士提供提示等），模型必须：
  - 低延迟地处理持续输入的视觉流；
  - 主动响应（proactiveness）当前场景变化；
  - 保持长期一致的语义输出（consistency）。  

- **核心问题**  
  当前缺乏一个能够真实模拟**视频流输入**、并纳入时间敏感评价标准的系统化评测框架。论文旨在填补这一空白，提出用于**实时流式视觉助手模型的统一评测标准与基准体系（Benchmark）**。

## 二、方法论概述

### 1. 核心思想
作者提出 **VSAS-Bench（Visual Streaming Assistant Benchmark）**，一个针对流式 VLM 的**实时评测框架**。其关键贡献包括：
- 引入 **同步（Synchronous）** 与 **异步（Asynchronous）** 两种评估协议；
- 建立时间密集型标注（每秒 1 帧）；
- 定义新型评测指标以度量模型在真实时间约束下的行为表现。

### 2. 评估协议与算法流程

#### （1）同步协议（Synchronous Protocol）
- 模型与视频帧的处理保持锁步。
- 模型无时间压力，可在任意延迟下处理视频帧。
- 适用于传统视频问答任务，但无法体现真实流处理需求。

#### （2）异步协议（Asynchronous Protocol）
- 视频流与模型推理的进程解耦，模拟**真实场景中的相机与模型处理速度不一致**。
- 采用 **摄像头缓冲区（Camera Buffer）** 存储即将处理的帧，并在模型空闲时取用。
- 若模型滞后，早期帧被丢弃，反映延迟对性能的真实影响。
- 伪代码（Algorithm 1）描述了生产者-消费者模式的运行逻辑。

### 3. 关键指标定义

- **平均准确率（Mean Average Accuracy, A）**  
  对每秒生成的模型响应 \( R_i \) 与标注 \( G_i \) 进行语义匹配，由 GPT-5 作为评审模型生成相似度分数。  

  \[
  A_t = \frac{1}{N_t}\sum_i Judge(G_i, R_i)
  \]

- **平均一致性（Mean Average Consistency, C）**  
  衡量模型跨时间步的稳定性，基于编辑距离或最长公共子串（LCS）法：

  \[
  C_t = 1 - D(R_i, R_{i+1}) + D(G_i, G_{i+1})
  \]

- 两指标在所有任务平均后得到整体性能。

### 4. 训练外适配（Training-free Adaptation）
- 提出一种**无需额外训练**的模型转化方式，使离线视频理解模型即可支持流式输入：
  - 引入 **模型记忆缓冲区（Model Memory Buffer）**；
  - 定义三种 **记忆策略（Memory Policy）**：
    - 滑动窗口（SW）；
    - 均匀采样（U）；
    - 滑动窗口 + 均匀尾部（SW+U）；
  - 实现**上下文拼接策略**，在保留时间连续性的同时保持推理高效。

## 三、实验设计与对比

### 1. 数据集与场景

- **VSAS-Bench 数据集**
  - 共计 **92 个视频**；
  - 包含 48 个新录制视频（包括室内活动、城市动作、UI 操作、烹饪任务等）；
  - 另有 44 个来自已有基准（STAR、Perception-Test、YouCook2）；
  - 总标注量超 **18,410 条**，时间分辨率为 **1 FPS**；
  - 任务类型分为：
    - **Present**（当前事件识别）；
    - **Cumulative**（累积过去信息推理）；
    - **Future**（预测未来事件）。

### 2. 对比方法

评估对象包括：
- **传统视频 VLMs**（如 Qwen3-VL、InternVL3 系列）；
- **流式专用 VLMs**（如 Dispider、VideoLLM-Online、FlashVStream、StreamBridge）；
- **闭源 API 模型**（如 GPT-5）。

### 3. 对比指标
- 两种协议下的 Accuracy、Consistency；
- 评估不同的缓冲区大小、分辨率、硬件配置；
- 平均延迟分析（End-to-End Latency）。

## 六、主要发现与结论

- 提出首个**时间密集、任务多样、响应自由格式**的流式视觉语言评测框架；
- 明确验证了：
  1. **异步评测**能正确揭示真实场景的延迟影响；
  2. **小模型更适合边缘设备的流式任务**（Qwen3-VL-4B 在异步场景表现最佳）；
  3. **无训练适配法（memory buffer）使传统视频模型优于专用流式模型**；
  4. **Dispider**在未来事件预测方面仍具竞争力，但整体性能不如 Qwen3-VL-4B；
  5. 提出 **Self-Speculative Decoding** 可降低延迟 33.6%，提升准确度与一致性。
