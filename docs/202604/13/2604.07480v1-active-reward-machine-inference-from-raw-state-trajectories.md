## 摘要
奖励机是一种类似自动机的结构，用于描述完成多阶段任务所需的记忆。当与强化学习或最优控制方法结合时，它们可以用于合成机器人策略以实现此类任务。然而，人工指定一个奖励机——包括捕捉决策所依据高级特征的标注函数——可能是一项艰巨的工作。本文研究直接从原始状态和策略信息学习奖励机的问题。与现有工作不同，我们假设无法获得奖励、标签或机器节点的观测，并展示在这种信息稀缺的条件下哪些轨迹数据足以学习奖励机。随后，我们将结果扩展到主动学习设置中，通过逐步查询轨迹扩展来提高数据（以及间接的计算）效率。我们用多个网格世界示例展示了结果。

## Abstract
Reward machines are automaton-like structures that capture the memory required to accomplish a multi-stage task. When combined with reinforcement learning or optimal control methods, they can be used to synthesize robot policies to achieve such tasks. However, specifying a reward machine by hand, including a labeling function capturing high-level features that the decisions are based on, can be a daunting task. This paper deals with the problem of learning reward machines directly from raw state and policy information. As opposed to existing works, we assume no access to observations of rewards, labels, or machine nodes, and show what trajectory data is sufficient for learning the reward machine in this information-scarce regime. We then extend the result to an active learning setting where we incrementally query trajectory extensions to improve data (and indirectly computational) efficiency. Results are demonstrated with several grid world examples.

## 二、方法论与技术创新

### 1. 整体思路

- 将**奖励机推断问题转化为可满足性（SAT）问题**，以逻辑约束的形式学习：
  - 奖励机的转移函数 δ_u；
  - 状态到原子命题的标注函数 L。

- 构建一个理论框架，证明存在一个足够的历史深度 \( l^* = |S| \cdot u_{max}^2 \)，保证只要在该深度下SAT问题可解，则更深层次同样成立。

### 2. 关键定义

- **历史策略（history policy）**  
  捕捉了当前状态与过去轨迹的依赖，用于描述智能体在仅观测状态时的行为分布。  
  \( \pi_h(a|s, \tau) = \pi_{Prod}(a|s, δ^*_u(u_I, L(\tau))) \)

- **负样本（negative examples）**  
  若两条轨迹导致不同动作分布（即智能体策略不同），则它们在奖励机中应对应不同节点。  
  这些负样本为SAT约束提供信息，限制可能的RM结构。

### 3. SAT 约束问题构建

- 引入布尔变量代表奖励机转移与标签映射：
  - \( b^p_{ji} = 1 \) 当且仅当 \( δ_u(i,p)=j \)
  - \( L^p_k = 1 \) 当且仅当 \( L(k)=p \)
- 约束包括：
  1. 转移函数唯一性；
  2. 标签函数定义性；
  3. 锚定（固定初始标签）；
  4. 满足所有负样本的区分性；
  5. 可选的“非重复自跳跃”约束以防过拟合。

- 通过求解此SAT问题，得到所有与观测数据一致的候选奖励机。

### 4. 主动扩展算法（Active Extension）

- 观察：不是所有轨迹都对缩小解空间有帮助。
- 提出主动策略：
  - **从已有候选解中生成候选轨迹对 {τ, τ′}**；
  - **定义“质量函数” quality(τ, τ′)** 测量其对分离解空间的潜力；
  - **优先查询**最能减少候选数量的一批轨迹；
  - **增量更新SAT问题**而非重新求解。

- **算法流程简述**：
  1. 初始化可行解集合；
  2. 随机采样部分候选模型；
  3. 用DFS生成可能引发节点冲突的轨迹对；
  4. 计算轨迹对的质量指标；
  5. 向环境查询有限预算B的高质量轨迹；
  6. 更新负样本并重新求解SAT；
  7. 若候选集收敛到唯一模型（或等价类）则停止。
