## 摘要
联邦学习（Federated Learning，FL）使多个参与方能够在不集中原始数据的情况下协作训练模型。FL 有两种主要范式：水平联邦学习（Horizontal FL，HFL），其中所有参与方共享相同的特征空间但拥有不同的样本；以及垂直联邦学习（Vertical FL，VFL），其中各方针对同一组样本拥有互补的特征。VFL 训练的一个前提是隐私保护实体对齐（Privacy-Preserving Entity Alignment，PPEA），即在不暴露哪些样本在各方之间共享的情况下（对齐），为样本建立共同索引。传统的私有集合求交（Private Set Intersection，PSI）能够实现对齐，但会泄露交集成员信息，从而暴露数据集之间的敏感关系。标准私有集合并集（Private Set Union，PSU）通过在标识符并集上对齐而非交集，从而降低这一风险。然而，现有方法通常仅限于两方场景或不支持拼写误差容忍匹配。本文提出了适用于 VFL 的 Sherpa.ai 多方 PSU 协议，这是一种隐藏交集成员信息并同时支持精确与噪声匹配的 PPEA 方法。该协议将两方方法推广至多方场景，同时保持较低通信开销，并提供两种变体：一种是保持顺序的版本，用于精确对齐；另一种是非顺序版本，可容忍拼写及格式差异。我们证明了其正确性与隐私性，分析了通信与计算（指数运算）复杂度，并形式化了从本地记录到共享索引空间的通用索引映射。该多方 PSU 协议为实际 VFL 部署（如多机构医疗疾病检测、银行与保险公司的协作风险建模，以及电信与金融机构间的跨领域欺诈检测）提供了一种可扩展、数学上稳健且能保护交集隐私的 PPEA 解决方案。

## Abstract
Federated Learning (FL) enables collaborative model training among multiple parties without centralizing raw data. There are two main paradigms in FL: Horizontal FL (HFL), where all participants share the same feature space but hold different samples, and Vertical FL (VFL), where parties possess complementary features for the same set of samples. A prerequisite for VFL training is privacy-preserving entity alignment (PPEA), which establishes a common index of samples across parties (alignment) without revealing which samples are shared between them. Conventional private set intersection (PSI) achieves alignment but leaks intersection membership, exposing sensitive relationships between datasets. The standard private set union (PSU) mitigates this risk by aligning on the union of identifiers rather than the intersection. However, existing approaches are often limited to two parties or lack support for typo-tolerant matching.   In this paper, we introduce the Sherpa.ai multi-party PSU protocol for VFL, a PPEA method that hides intersection membership and enables both exact and noisy matching. The protocol generalizes two-party approaches to multiple parties with low communication overhead and offers two variants: an order-preserving version for exact alignment and an unordered version tolerant to typographical and formatting discrepancies. We prove correctness and privacy, analyze communication and computational (exponentiation) complexity, and formalize a universal index mapping from local records to a shared index space. This multi-party PSU offers a scalable, mathematically grounded protocol for PPEA in real-world VFL deployments, such as multi-institutional healthcare disease detection, collaborative risk modeling between banks and insurers, and cross-domain fraud detection between telecommunications and financial institutions, while preserving intersection privacy.

# Sherpa.ai 隐私保护多方实体对齐研究总结

## 一、核心问题与研究背景

- **研究主题**：本文研究纵向联邦学习（Vertical Federated Learning, VFL）中的**隐私保护实体对齐**（Privacy-Preserving Entity Alignment, PPEA）问题。  
- **问题背景**：  
  - 在 VFL 场景下，各参与方持有同一实体（如同一客户或患者）的不同特征，但在联合训练前需确定哪些样本属于相同实体，这一过程称为实体对齐。  
  - 传统的**私有集合求交**（Private Set Intersection, PSI）方法虽可实现对齐，但会泄露交集成员信息，带来敏感隐私风险。  
  - **私有集合并集**（Private Set Union, PSU）可在并集层面对齐，从而避免暴露共享样本，但现有方法通常：
    - 仅支持**两方对齐**；
    - 不具备容错性（文字拼写、格式差异会导致匹配失败）。  
- **研究目标**：提出一种适用于多方的、兼顾隐私性与匹配鲁棒性的 PSU 协议，用于真实世界多机构 VFL 场景，如医疗、金融与通信等跨域协作。

## 二、方法论与关键技术

### 2.1 核心思想

- **目标**：在不暴露交集成员信息的情况下，实现多个参与方之间的样本对齐。  
- **核心机制**：基于**Diffie–Hellman 可交换加密（commutative encryption）**和**哈希化 n-gram 标识符表示**，实现多方 PSU。  
- **两种对齐模式**：
  1. **有序（Exact / Order-Preserving）版本**：适用于干净、格式统一的数据，保证精确匹配；
  2. **无序（Noisy / Unordered）版本**：支持对字符串扰动、拼写错误、不同分词格式的鲁棒匹配。

### 2.2 关键技术流程

#### (1) n-gram 标识符分词  
- 将标识符（如姓名、地址、电话）拆分为长度为 *n* 的重叠子串。
- 目的：增强对格式差异与随机噪声的容忍性。
- 例如 `"123 Main St"` 与 `"123 Main Street"` 在多数 n-gram 上仍存在重叠。

#### (2) 哈希与加密  
- 使用 **SHA3-256 哈希函数**将 n-gram 映射为固定长度的整数；
- 映射到安全素数模群 \( QR(Z_p^*) \)；
- 每方使用私有指数 \( s_k \)，执行加密函数：
  \[
  e_s(x) = x^s \mod p
  \]
  加密具有可交换性：\( e_{s1}(e_{s2}(x)) = e_{s2}(e_{s1}(x)) \)。

#### (3) 多方 PSU 协议
- 核心分为三个阶段：

  1. **第一阶段：多方交换与加密**
     - 各方对其哈希后的标识符进行加密、随机打乱，并在多方之间循环交换。
  2. **第二阶段：并集构造**
     - “主动方”整合所有经双重加密的集合，计算总体并集；
     - 再次加密形成最终加密标识符集 \(U\)，广播给所有参与方。
  3. **第三阶段：匹配与索引分配**
     - 各方本地再加密自身标识符，并在 \(U\) 中查找对应位置；
     - 建立从本地索引到通用索引的映射 \( \phi_k \)。

#### (4) 无序版本的扩展机制
- 随机打乱标识符的 n-gram 顺序；
- 使用**匹配阈值 λ** 判断两个加密后的标识符是否近似匹配；
- 引入 **Bloom filter** 以降低存储与比较成本；
- 可在实际实现中根据领域（如姓名、地址）调整 λ。

#### (5) 缺失数据填补
- 对未对齐样本，使用 **Synthetic Data Vault (SDV)** 工具生成合成样本；
- 提供两种后端：
  - **Gaussian Copula**：适用于中等维度数据；
  - **CTGAN**：用于分布复杂或类别不平衡的场景。
