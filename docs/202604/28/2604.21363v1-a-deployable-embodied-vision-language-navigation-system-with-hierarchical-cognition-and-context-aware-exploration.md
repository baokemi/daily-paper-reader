## 摘要
在智能机器人系统中，弥合具身智能与嵌入式部署之间的差距仍然是关键挑战之一，因为感知、推理和规划必须在计算、内存、能耗和实时执行等严格约束下运行。在视觉语言导航（Vision-Language Navigation, VLN）领域，现有方法往往面临强推理能力与现实平台高效部署之间的根本权衡。本文提出了一种可部署的具身VLN系统，能够在真实机器人平台上同时实现高效率和稳健的高级推理。为此，我们将系统分解为三个异步模块：实时感知模块用于持续环境感知，记忆整合模块用于空间-语义聚合，推理模块用于高级决策。我们逐步构建认知记忆图以编码场景信息，并进一步将其分解为子图，以支持利用视觉语言模型（VLM）进行推理。为进一步提升导航效率与准确性，我们还利用认知记忆图将探索问题建模为具有上下文感知的加权旅行修理工问题（Weighted Traveling Repairman Problem, WTRP），以最小化视点的加权等待时间。大量在仿真和真实机器人平台上的实验表明，该系统在保持资源受限硬件实时性能的同时，相较现有VLN方法显著提升了导航成功率与效率。

## Abstract
Bridging the gap between embodied intelligence and embedded deployment remains a key challenge in intelligent robotic systems, where perception, reasoning, and planning must operate under strict constraints on computation, memory, energy, and real-time execution. In vision-language navigation (VLN), existing approaches often face a fundamental trade-off between strong reasoning capabilities and efficient deployment on real-world platforms. In this paper, we present a deployable embodied VLN system that achieves both high efficiency and robust high-level reasoning on real-world robotic platforms. To achieve this, we decouple the system into three asynchronous modules: a real-time perception module for continuous environment sensing, a memory integration module for spatial-semantic aggregation, and a reasoning module for high-level decision making. We incrementally construct a cognitive memory graph to encode scene information, which is further decomposed into subgraphs to enable reasoning with a vision-language model (VLM). To further improve navigation efficiency and accuracy, we also leverage the cognitive memory graph to formulate the exploration problem as a context-aware Weighted Traveling Repairman Problem (WTRP), which minimizes the weighted waiting time of viewpoints. Extensive experiments in both simulation and real-world robotic platforms demonstrate improved navigation success and efficiency over existing VLN approaches, while maintaining real-time performance on resource-constrained hardware.

## 二、方法论与核心技术

### 1. 系统总体架构：**层次认知与异步设计**

- **三层异步模块架构**：
  1. **实时感知层（Perception Layer）**：负责20 Hz高频感知、定位、避障与局部规划；
  2. **记忆构建层（Memory Layer）**：增量式构建**认知记忆图**（Cognitive Memory Graph, CMG），组织场景的空间与语义信息；
  3. **高层推理层（Reasoning Layer）**：低频（1–2 Hz）运行，对CMG进行子图分解并利用VLM（Qwen3-Omni）进行推理与决策。

- **模块并行与异步通信**确保在嵌入式设备上同时实现响应性和推理能力。

### 2. 认知记忆图（Cognitive Memory Graph, CMG）

- **图结构定义**：双部图 \( G = (V_a, V_o, E) \)，其中  
  - \(V_a\)：视觉锚节点（代表关键帧）；  
  - \(V_o\)：对象节点；  
  - \(E\)：观测关系（anchor与object的连接）。
- **构建流程**：
  - 使用 **YOLO-World** 检测开放词汇目标；
  - 应用 **Mobile-SAM** 获取实例分割；
  - 用LiDAR-SLAM提取位姿，计算3D点云；
  - 通过**几何IoU+CLIP语义相似度**融合冗余对象节点；
  - 最终选择性保留锚点，使用整数线性规划（Weighted Set Multicover Problem）实现内存压缩与语义完备性平衡。

### 3. 异步深度推理模块

- 按任务指令将CMG分解为多个**子图（subgraphs）**：
  - **目标子图 \(G_t\)** 与**关联子图 \(G_r\)** 聚焦于目标及其语义上下文；
  - 其余区域子图则基于CLIP相似度打分决定优先级。
- **推理过程**：
  - 通过Qwen3-Omni API异步调用VLM；
  - 对每一子图顺序判断目标可见性→定位关键锚点→输出语义证据；
  - 实现推理与实时控制的解耦。

### 4. 上下文感知探索策略

- **多源信息融合的前沿打分**：
  \[
  S(f_i) = \lambda_1 S_{struct}(f_i) + \lambda_2 S_{vis}(f_i)
  \]
  - 结构语义分数 \(S_{struct}\)：模拟对象语义场的空间衰减影响；
  - 视觉证据分数 \(S_{vis}\)：通过CLIP计算边界图像与任务语言的匹配。
- **任务先验调制**：VLM分解出的任务相关对象增强其语义权重。

### 5. 目标选择与规划（WTRP优化）

- 将探索建模为**加权旅行维修员问题（Weighted Traveling Repairman Problem, WTRP）**：
  \[
  \min_\pi \sum_i W_i C_i(\pi)
  \]
  其中 \(W_i\) 由前沿语义打分归一化后指数加权，
  \(C_i(\pi)\) 为访问顺序累计代价。
- **过渡成本模型**同时考虑位移、转向约束与局部结构封闭度，利用LKH启发式求解。
- **局部规划**：结合A*和动态障碍避让生成平滑轨迹。

## 八、不足与局限

- **三维空间限制**：仅支持2D地图探索，跨楼层（vertical navigation）任务失败率高；
- **视觉误检问题**：存在CLIP相似度误判导致“假阳性”；
- **小物体识别不稳定**：受运动模糊与分辨率影响；
- **算力依赖**：虽然采用异步优化，但VLM调用仍需外部API支持，离线本地化成本较高；
