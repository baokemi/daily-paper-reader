## 摘要
我们研究了近期提出的视觉-语言模型（VLMs）的领域-类别增量学习场景。现有工作通常采用参数高效的方法，例如前缀微调（prefix-tuning）或适配器（adapters），通过在输入 token 上添加向量，将任务相关信息融入模型，从而促进模型对下游任务的适应。然而，以往的方法常常对这些向量的权重进行归一化，而忽略了不同输入 token 需要不同程度的调整。为解决这一问题，我们提出了动态前缀加权（Dynamic Prefix Weighting，DPW）框架，该框架通过与适配器结合，动态为前缀分配权重。DPW 包含：(1) 一个基于对应输入 token 重要性调整各前缀权重的门控模块，以及 (2) 一个将适配器输出权重作为前缀微调权重残差的加权机制，确保适配器仅在必要时被使用。实验结果表明，我们的方法在视觉-语言模型的领域-类别增量学习场景中实现了当前最优性能。代码可在以下地址获取：https://github.com/YonseiML/dpw。

## Abstract
We investigate recently introduced domain-class incremental learning scenarios for vision-language models (VLMs). Recent works address this challenge using parameter-efficient methods, such as prefix-tuning or adapters, which facilitate model adaptation to downstream tasks by incorporating task-specific information into input tokens through additive vectors. However, previous approaches often normalize the weights of these vectors, disregarding the fact that different input tokens require different degrees of adjustment. To overcome this issue, we propose Dynamic Prefix Weighting (DPW), a framework that dynamically assigns weights to prefixes, complemented by adapters. DPW consists of 1) a gating module that adjusts the weights of each prefix based on the importance of the corresponding input token, and 2) a weighting mechanism that derives adapter output weights as a residual of prefix-tuning weights, ensuring that adapters are utilized only when necessary. Experimental results demonstrate that our method achieves state-of-the-art performance in domain-class incremental learning scenarios for VLMs. The code is available at: https://github.com/YonseiML/dpw.

## 一、研究动机与问题背景  

### 1. 持续学习的挑战  
- 持续学习（Continual Learning, CL）旨在让模型在遇到新的任务时，不遗忘旧任务知识，即缓解**灾难性遗忘**。  
- 对于大规模视觉-语言模型（VLMs，如 CLIP），全参数微调代价高昂，不适合在多领域顺序学习场景下使用。  

### 2. 参数高效微调（PEFT）方法的普及与问题  
- 常见做法包括 **前缀微调（Prefix-Tuning）** 与 **适配器（Adapters）**，通过冻结骨干网络，仅引入少量可学习参数以适配新任务。  
- 现有方法普遍采用**样本级统一权重**，忽略了不同输入 token 的任务相关性差异。  
- 如图示 (Fig. 1)，MoE-Adapters 与 DIKI 都对所有 token 施加均匀或归一化权重，导致信息注入过度或不足，降低 CL 适配效果。

### 3. 核心研究问题  
> 如何在视觉-语言模型的持续学习中，实现对**token 级别的任务相关性自适应控制**，平衡前缀信息与适配器更新，从而在保留基础知识的同时高效适应新任务？

## 二、方法论：动态前缀加权（Dynamic Prefix Weighting, DPW）

DPW 是一种面向持续学习场景的参数高效框架，核心思想是在 prefix-tuning 的基础上引入**动态权重门控机制**与**残差加权机制**，使模型能根据每个输入 token 的重要性动态分配前缀和适配器的权重。

### 1. 模型总体框架  
由三部分组成：
1. **Reparameterized Prefix Attention (RePA)**：重新参数化的前缀注意力  
2. **Conditional Activation (CondAct)**：基于条件激活的权重过滤  
3. **Residual Weighting Mechanism (RWM)**：残差加权的适配器整合机制

### 2. 核心技术要点  

#### （1）RePA：重参数化前缀注意力  
- 用可学习的仿射变换代替原有的 Query-Key 点积，直接从输入 token 计算前缀相关性得分。
- 目的是避免预训练注意力中“全局化”导致的任务判别信息模糊问题。
- 数学形式：  
  \[
  S_{XP}^{(i)} = XW_G^{(i)} + B_G^{(i)}
  \]
  通过训练 \(W_G, B_G\) 可获得任务相关性更高的前缀打分矩阵。

#### （2）CondAct：条件激活机制  
- 替换传统 Softmax，包含两步：  
  1. **条件归一化**：限制每个 token 的前缀总权重和不超过 1，防止过度任务特化。  
  2. **条件过滤**：基于 prefix-[CLS] 注意分布的高斯似然动态阈值，过滤无关前缀。  
- 从而在保持零样本泛化的同时，引入 token 级动态调节（Eq. 8–9）。

#### （3）RWM：残差加权适配器  
- 当前缀权重不足时，通过残差部分激活适配器输出。  
- 适配器采用改进的 **HydraLoRA** 结构：共享下投矩阵 \(D\)（经 SVD 初始化并冻结），仅学习上投矩阵 \(U_t\)。  
- 数学表达：
  \[
  O_{\text{adapter}}^{(i)} = \Delta^{(i)} \odot E_t^{(i)}(X), \quad \Delta_{ij} = \max(0, \sum_k \sigma(s_{ijk}) - 1)
  \]
  表示当 prefix 不足时，残差部分驱动适配器补充调整。

## 三、实验设计  

### 1. Benchmark 数据集与场景  
- **MTIL（Multi-domain Task Incremental Learning）**：具任务 ID 的跨领域增量学习。  
- **ODCL-CIL（Open-Domain Class-Incremental Learning）**：无任务 ID 的类别增量学习。  
- 包含 **11 个数据集**，如 Aircraft、Caltech101、CIFAR100、Flowers、Food、OxfordPet 等，覆盖 1201 个类别。

### 2. 评估指标  
- **Transfer**：衡量前向遗忘（Forward Forgetting）；  
- **Avg**：所有阶段平均准确率；  
- **Last**：模型在最终任务的平均准确率。

### 3. 对比方法  
- **全参数微调类**：ZSCL、GIFT  
- **适配器类**：MoE-Adapters  
- **前缀类**：DIKI、CoLeCLIP、DPeCLIP  
- **本文方法**：DPW（完整版本）和 DPW†（参数更高效版本）

### 4. 超参数与实现细节  
- 视觉-语言基座：CLIP ViT-B/16

## 五、主要结论与发现  

- DPW 在 **MTIL** 和 **ODCL-CIL** 两个场景上均达到了 **SOTA** 性能：  
  - **MTIL 平均分提升约 2.0%，ODCL-CIL 提升约 2.5%**。  
- 相比 DIKI、MoE-Adapters 等基线，DPW 在仅增加 1/3 参数量的情况下显著提升泛化与记忆保留能力。  
- 三个子模块协同效果明显：RePA 改善任务相关性，CondAct 提升零样本稳定性，RWM 提升后期适应力。  
- DPW 实现了 **token 级自适应加权机制**，使得模型在已学任务依赖前缀，在新任务中自动增强适配器响应。  
- 理论上证明 CondAct 能将前缀引入的表示漂移上界从 \(Lc_V\) 控制至 \(c_V\)，有效减少前向遗忘。

## 六、优点与创新亮点  

- ✅ **创新架构设计**：首次实现前缀与适配器的动态融合，兼容多模态结构。

- ✅ **参数效率优越**：DPW† 仅使用 ~4.6M 可训练参数即可超越基线（DIKI 的 1.8M 参数）。

## 七、局限与不足  

- ⚠ **实验集中于 CLIP 模型**：未验证在其他大型预训练模型（如 BLIP2、ALIGN）上的泛化性。  
- ⚠ **任务场景相对受限**：仅评估领域-类别增量场景，缺少时间序列或多模态交互任务。  
- ⚠ **超参数敏感性分析不足**：动态阈值中高斯参数选择主要基于经验。  
- ⚠ **实用性验证缺乏**：未展示 DPW 在真实持续在线学习或数据漂移环境下的效果。  
- ⚠ **理论部分简化假设较强**：假设前缀分布与 [CLS] 注意呈高斯分布，可能不完全成立于复杂视觉语义任务。
