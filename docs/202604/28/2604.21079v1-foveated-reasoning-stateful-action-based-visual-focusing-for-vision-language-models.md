## 摘要
视觉-语言模型受益于高分辨率图像，但视觉标记数量的增加会带来高计算开销。人类通过中心凹化（foveation）来解决这一矛盾：通过粗略视图决定“看向何处”，而通过选择性地获取高精度信息来改进“思考内容”。我们提出了Foveated Reasoner，这是一种自回归的视觉-语言框架，将中心凹化与推理统一到同一个解码轨迹中。模型从低分辨率视图开始，仅在需要时触发中心凹化，从选定区域提取高分辨率证据，并将其重新注入同一解码轨迹。我们采用两阶段训练流程：首先通过冷启动监督来引导中心凹化行为，然后通过强化学习共同改进证据获取与任务准确性，同时抑制琐碎的“看遍一切”式解决方案。实验结果表明，该方法能够学习有效的中心凹化策略，并在多项视觉-语言基准测试中，在严格的视觉标记预算下取得了更高的准确率。

## Abstract
Vision-language models benefit from high-resolution images, but the increase in visual-token count incurs high compute overhead. Humans resolve this tension via foveation: a coarse view guides "where to look", while selectively acquired high-acuity evidence refines "what to think". We introduce Foveated Reasoner, an autoregressive vision-language framework that unifies foveation and reasoning within a single decoding trajectory. Starting from a low-resolution view, the model triggers foveation only when needed, retrieves high-resolution evidence from selected regions, and injects it back into the same decoding trajectory. We train the method with a two-stage pipeline: coldstart supervision to bootstrap foveation behavior, followed by reinforcement learning to jointly improve evidence acquisition and task accuracy while discouraging trivial "see-everything" solutions. Experiments show that the method learns effective foveation policies and achieves stronger accuracy under tight visual-token budgets across multiple vision-language benchmarks.

## 一、研究背景与核心问题

- **问题动机**：  
  视觉语言模型（VLMs）在处理高分辨率图像时能获得更好的理解与推理能力，但视觉标记（visual tokens）数量随分辨率平方增长，使得计算和内存开销急剧增加。  
  人类视觉系统通过“中心凹化（foveation）”机制平衡效率与准确性：低精度外周视用于指导“看向何处”，高精度局部视用于获取关键细节。因此，作者希望在人工模型中复现这种 **状态保持的、动作驱动的视觉聚焦过程**，以减少计算代价同时保持推理连续性。

- **核心挑战**：  
  现有视觉聚焦策略存在两种主流但均有限制：
  1. **多次运行（multi-pass）方法**：多次推理、打断状态连续性；
  2. **文本驱动（text-grounded）方法**：通过语言坐标表达聚焦，带来额外 token 消耗和格式脆弱性。  
  两者均无法实现人类视觉那样的连续、非语言化的注视过程。

- **研究目标**：  
  构建一种单次自回归的视觉语言模型，使其能在同一解码轨迹中动态执行“聚焦动作（foveation）”，即基于当前推理状态自适应地决定是否采集新的高分辨率视觉证据。

## 二、方法论与核心模型设计

### 1. 整体思想
- 提出 **Foveated Reasoner (FoveateR)**：将视觉-语言推理与中心凹化机制统一到一个连续的自回归生成过程中。  
- 其核心思想是将模型视作一个在部分可观察环境中决策的智能体。模型从低分辨率图像开始，仅在“需要时”触发 foveation 动作， 获取高分辨率区域特征并将其直接注入当前的推理状态中。

### 2. 方法模型化
- **形式化为部分可观测马尔可夫决策过程（POMDP）**：
  - 环境状态：包含低分辨率输入和底层高分辨率原图。
  - 动作空间：  
    - 离散动作：生成下一个文本 token；  
    - 连续动作：由 foveation policy 预测聚焦区域框 `[cx, cy, w, h]`。
  - 记忆机制：Transformer 的 KV-cache 存储所有历史 token 及视觉证据，保证状态连续性。
  - 状态表示：Decoder 隐状态向量 `h_t = f_θ(M_t)` 表示智能体的“信念”或观察整合结果。

### 3. 动作与联合策略
- 模型在解码过程中根据当前隐状态决定：
  ```
  y_t ∼ π_θ(·|h_t),  y_t ∈ V ∪ {<fov>}
  ```
  若输出 `<fov>`，则调用额外的策略网络 π_ϕ 预测连续区域框，并提取视觉块：
  ```
  b_t ∼ π_ϕ(·|h_t) -> V_t = Tokenize(Crop(I, b_t))
  ```
  再将高分辨率证据 `<fov>V_t</fov>` 直接注入生成流中，无需重新初始化模型。

- **特性**：
  - 单次自回归轨迹（无重复运行）；  
  - 在推理和感知之间保持共享状态；  
  - 以非语言动作实现空间聚焦，无坐标文本化。

### 4. 训练流程

#### （1）冷启动监督学习（Coldstart SFT）
- **目标**：引导模型学会何时触发 `<fov>` 与预测盒子区域。
- **策略**：
  - 基于 Visual CoT 等数据，生成伪监督序列：  
    在每一步推理文本前插入相应的 `<fov>` 高分辨率证据块。  
  - 优化目标：
    ```
    L_cold = L_LLM + λ_box L_box
    ```
    其中 L_LLM 为交叉熵损失，L_box 为框预测 L1 损失。

#### （2）强化学习（RL）阶段
- **算法**：使用 Group-Relative Policy Optimization (GRPO)，类似 PPO。  
- **奖励设计**：
  - 准确性奖励 `r_acc`: 最终答案正确性；
  - 格式奖励 `r_fmt`: 合规输出结构；
  - 正则项 `L_reg`: 惩罚过大的聚焦区域，防止“看遍全图”。
- 最终优化式：
  ```
  L_RL = L_GRPO-tok + λ_fov L_GRPO-fov + λ_reg L_reg
  ```

## 六、主要结论与发现

1. **性能提升显著**：  
   FoveateR 在大多数任务上较多次聚焦或文本标注方法有明显优势。  
   - 336² 设置下，3B 模型平均 token 数（307）远低于对比模型（>1100），但准确率更高。  
   - 在高分辨率 V*Bench 上，小规模模型仍超越 7B–12B 竞争者。

2. **学习到有效策略**：  
   模型根据任务自动调整聚焦次数与区域所占比例，在简单任务中无需聚焦，而复杂文档任务中触发多次局部放大。

3. **伪监督优于人工标注**：  
   使用 VLM 自动生成的伪框监督能更好学习动态、多步注视行为。

4. **具备状态连续性与效率平衡**：  
   自回归过程中不重新初始化隐藏状态，实现连贯的视觉推理。

## 八、不足与局限性

- **计算增长风险**：随着多次聚焦累积，高分辨率 token 注入 KV-cache 使显存线性增长。
- **实验范围限制**：仅评估单图任务，未验证视频或长序列场景中的时序聚焦能力。
- **策略空间有限**：当前聚焦区域仅单框 `[cx, cy, w, h]`，无法同时获取多区域或变分分辨率。
- **潜在偏差**：伪监督依赖预训练模型生成标注，可能引入训练分布偏移。
- **实际部署挑战**：RL 阶段仍需大量算力，对移动端或低功耗应用的落地还有距离。
