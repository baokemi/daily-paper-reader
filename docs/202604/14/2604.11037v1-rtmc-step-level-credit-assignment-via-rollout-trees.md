## 摘要
多步代理强化学习从精细化的信用分配中获益，但现有方法的选择有限：无评论家（critic-free）的方法如 GRPO 为轨迹中的每个动作分配相同的优势值，而学习到的价值网络会引入显著的开销，并且在稀疏奖励环境下可能不稳定。我们观察到，针对相同问题的组滚动（group rollouts）通常会经过重叠的中间状态，从而隐式地形成一棵树，其分支在连续的决策点处发散。基于这一洞见，我们提出了 Rollout-Tree Monte Carlo（RTMC）优势估计方法，它在没有学习评论家的情况下，通过聚合跨滚动轨迹共享同一状态的回报统计，生成逐步的 Q 值和优势。一个状态-动作签名系统将原始交互历史压缩为紧凑且可比的表示，使跨滚动的状态匹配变得可行。在 SWE-bench Verified 上，RTMC 相较于 GRPO 将 pass@1 提升了 3.2 个百分点。

## Abstract
Multi-step agentic reinforcement learning benefits from fine-grained credit assignment, yet existing approaches offer limited options: critic-free methods like GRPO assign a uniform advantage to every action in a trajectory, while learned value networks introduce notable overhead and can be fragile under sparse rewards. We observe that group rollouts targeting the same problem often traverse overlapping intermediate states, implicitly forming a tree whose branches diverge at successive decision points. Building on this insight, we introduce Rollout-Tree Monte Carlo (RTMC) advantage estimation, which aggregates return statistics across rollouts sharing a common state to produce per-step Q-values and advantages--without any learned critic. A state-action signature system compresses raw interaction histories into compact, comparable representations, making cross-rollout state matching tractable. On SWE-bench Verified, RTMC improves pass@1 by 3.2 percentage points over GRPO.

- **核心问题**：在无需学习价值网络的情况下，实现**高效、细粒度的逐步信用分配**，以提升多步强化学习在 LLM 代理任务中的稳定性和性能。

## 二、方法论：Rollout-Tree Monte Carlo (RTMC)

### 1. 核心思想
- 将同一任务的多条滚动轨迹（rollouts）视为**隐式形成的树结构**：  
  当不同轨迹在某些中间状态重叠时，它们可共享经验。  
- **利用跨轨迹统计信息**，在每个状态聚合其后续回报，从而估算每步的 Q 值与优势，而不必训练评论家。

### 2. 技术关键
- **状态-动作签名系统（State-Action Signatures）**：
  - 压缩原始交互历史为可比较的标识字符串。
  - 支持跨滚动匹配“相同”状态，即使文本上下文不同。
  - 在代码任务中通过分类（view、search、modify、create 等）、哈希和分桶实现。

- **Monte Carlo优值估计**：
  - 对于每个状态-动作对 \((s,a)\)，计算：
    \[
    \hat{Q}(s,a) = \frac{1}{N(s,a)}\sum_i G_t^{(i)}
    \]
    \[
    \hat{V}(s) = \frac{\sum_a N(s,a)\hat{Q}(s,a)}{\sum_a N(s,a)}
    \]
    \[
    \hat{A}(s,a) = \hat{Q}(s,a) - \hat{V}(s)
    \]
  - 实现**逐步优势分配**，将每步的优势广播到对应token段。

- **算法流程**：
  1. **阶段一：树构建**  
     - 遍历每条轨迹，生成状态签名与动作签名，计算折扣回报并填入统计表。
  2. **阶段二：优势计算**  
     - 对统计表做均值估计，计算每步 \(\hat{A}\)，并分配到token。  
  3. **时间复杂度**：线性 \(O(N·L)\)，无需额外梯度或再推理。

- **改进机制**：
  - **先验平滑（Prior-based Value Smoothing）**：  
    在稀疏访问状态下混合全局成功率作为贝叶斯缩减项。
  - **任务特定奖励修正**：  
    对代码验证步骤加微量奖励避免 γ 折扣导致惩罚。
