## 摘要
大型语言模型（LLMs）具有概率特性，并且在与外部信息结合时表现得更为可靠。然而，复杂查询通常需要对检索到的信息进行多步推理，缺乏明确或预设的推理路径，因此仍然具有挑战性。近期的方法通过对模型输出使用强化学习进行训练，在改进模型处理复杂信息的能力方面表现出潜力。我们提出了 SubSearch，这是一种专为将监督从仅基于结果的信号转向中间奖励信号而设计的框架，以激励规划高质量的推理。不同于以往的过程奖励建模方法，这些方法依赖人工标注或大型 LLM 评审生成的带注释轨迹来训练独立的奖励模型，SubSearch 直接利用内在过程奖励（定义为模型内部产生的奖励）优化生成器，从而无需外部监督，迈向自主的信息密集型推理。基于七个基准的实验表明，与仅使用结果奖励相比，对中间推理步骤给予内在奖励可以在 QA 和多跳 QA 数据集上产生更稳健的推理轨迹。SubSearch 有助于构建推理轨迹，使智能体能够更好地整合搜索引擎以回答复杂查询，同时提供了一种相比监督式过程建模更高数据效率的替代方案。

## Abstract
Large language models (LLMs) are probabilistic in nature and perform more reliably when augmented with external information. As complex queries often require multi-step reasoning over the retrieved information, with no clear or predetermined reasoning path, they remain challenging. Recent approaches train models using reinforcement learning on the model's outcome, showing promise in improving how models handle complex information. We introduce SubSearch, a specialized framework that shifts from outcome-only supervision to intermediate reward signals that incentivize planning high-quality reasoning. Unlike previous work on process reward modeling, which focuses on training a separate reward model with annotated trajectories by either human annotators or large LLM judges, SubSearch directly optimizes the generator using intrinsic process rewards, which we define as internally-derived rewards, eliminating the need for external supervision, and moving towards autonomous information-intensive reasoning. Experiments on seven benchmarks show that rewarding intermediate reasoning steps with intrinsic rewards leads to more robust reasoning traces in both QA and multi-hop QA datasets over using only outcome rewards. SubSearch can help in building reasoning traces that allow agents to better integrate search engines for complex query answering, while offering a data-efficient alternative to supervised process modeling.

## 一、研究核心问题与背景

- **研究动机**：复杂信息检索任务（如多跳问答）要求模型执行**多步推理**和信息整合，而大型语言模型（LLMs）在没有明确推理路径和外部标注监督时，常表现出推理不稳定或“投机取巧”（reward hacking）。
- **现有问题**：  
  - 目前的强化学习优化（RLHF 或 RLVF）大多依赖**仅基于最终答案的稀疏奖励（outcome reward）**；  
  - 这会导致模型虽然能输出正确答案，但中间推理过程可能错误或缺乏可解释性。  
- **研究目标**：作者希望**通过在推理过程中引入中间奖励信号（intermediate rewards）**来引导 LLM 学习更加稳健、可解释、可泛化的推理轨迹，且该过程无需人工标注或外部评判模型。

## 二、方法论：SubSearch 框架与关键机制

### 1. 核心思想
- SubSearch 是一种专门针对“复杂检索任务的多步推理”而设计的**强化学习框架（RL for Retrieval-Augmented Reasoning）**；
- 与传统过程奖励模型（PRM）区别在于：
  - PRM 需训练一个独立奖励模型并依赖外部监督；
  - SubSearch **直接在生成器（policy LLM）上施加“内生”中间奖励信号（intrinsic process reward）**，不依赖外部人工或LLM评审。

### 2. 框架组成
- **推理轨迹（Reasoning Trace）结构化表示**：
  - 使用标记包裹不同阶段：  
    `<think>…</think>`：推理思考；  
    `<search>…</search>`：生成搜索查询；  
    `<information>…</information>`：检索结果；  
    `<answer>…</answer>`：最终答案。  
- 策略模型通过这些标记循环执行“推理–检索–聚合”的多轮迭代，直至积累足够信息回答问题。

### 3. 关键奖励设计
SubSearch 结合多个层次的奖励信号，共同组成最终优化目标：
- **(a) 最终答案奖励（R_answer）**：  
  基于 Exact Match（EM）评分，与金标准答案是否匹配。
- **(b) 子查询可回答性奖励（R_answerability）**：  
  衡量子查询与检索结果的语义覆盖度，通过子查询与前 k 个检索文档向量的余弦相似度平均值计算。
- **(c) 查询分解奖励（R_decomposition）**：  
  同时约束两方面：
  - *语义覆盖度*（子查询语义应覆盖父查询信息）；  
  - *信息独立性*（各子查询之间应独立、互补）。  
  计算方式为两个相似度目标的加权组合。
- **(d) 格式奖励（r_format）**：  
  用于稳定训练，确保输出符合模板格式。
- **(e) 奖励聚合策略**：  
  提出“**自适应残差奖励聚合（adaptive residual aggregation）**”，动态平衡最终答案奖励与中间信号，防止过度惩罚有潜力但暂未收敛的路径。

### 4. 强化学习优化
- 使用 **Group Relative Policy Optimization (GRPO)** 算法（类似 PPO 的变体），并采用群组归一化的优势估计，以稳定学习过程。

## 六、主要结论与发现

- SubSearch 在所有基准上均优于仅基于结果奖励的强化学习方法：
  - 相比 Search-R1-base，在 **HotpotQA 提升 +6.5 EM，2WikiMultiHopQA 提升 +7.7 EM**；
  - 在复杂多跳问答上优化尤为显著；
- 引入中间奖励信号能显著提高：
  - 推理路径的可解释性与稳健性；
  - 模型在信息稠密任务中的泛化性；
- 无需任何外部标注或额外训练数据，即可实现接近甚至超过 SFT+RL 方法的表现；
- 自适应残差奖励机制在训练稳定性与效率上最优。
