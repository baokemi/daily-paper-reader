## 摘要
大规模视觉-语言模型（Large Vision-Language Models, LVLMs）能够在图像和视频上执行复杂的推理任务，但其推理效率受到一种被称为视觉标记主导（visual token dominance）的系统性瓶颈限制。这一开销源于高分辨率特征提取、平方级注意力扩展以及内存带宽约束之间的多重机制交互。本文提出了一个围绕推理生命周期构建的系统化效率技术分类法，包括编码、预填充（prefilling）和解码阶段。不同于以往关注孤立优化方法的综述，我们分析了端到端的处理流程，揭示了上游决策如何决定下游瓶颈，涵盖了计算受限的视觉编码、大规模上下文的高强度预填充，以及带宽受限解码过程中的“视觉记忆墙”。通过将效率空间分解为信息密度塑造、长上下文注意力管理与突破内存限制三个维度，本文对各类孤立优化方法如何协同作用以平衡视觉保真度与系统效率进行了系统分析。最后，我们基于初步实证见解，提出了四个未来发展方向：基于功能单元敏感度的混合压缩、具模态感知的宽松验证解码、面向流式连续性的渐进状态管理，以及通过软硬件协同设计实现的阶段解耦式服务架构。我们的文献资源库可在 https://github.com/SuDIS-ZJU/Efficient-LVLMs-Inference 获取。

## Abstract
Large Vision-Language Models (LVLMs) enable sophisticated reasoning over images and videos, yet their inference is hindered by a systemic efficiency barrier known as visual token dominance. This overhead is driven by a multi-regime interplay between high-resolution feature extraction, quadratic attention scaling, and memory bandwidth constraints. We present a systematic taxonomy of efficiency techniques structured around the inference lifecycle, consisting of encoding, prefilling, and decoding. Unlike prior reviews focused on isolated optimizations, we analyze the end-to-end pipeline to reveal how upstream decisions dictate downstream bottlenecks, covering compute-bound visual encoding, the intensive prefilling of massive contexts, and the ''visual memory wall'' in bandwidth-bound decoding. By decoupling the efficiency landscape into the axes of shaping information density, managing long-context attention, and overcoming memory limits, this work provides a structured analysis of how isolated optimizations compose to navigate the trade-off between visual fidelity and system efficiency. The survey concludes by outlining four future frontiers supported by pilot empirical insights, including hybrid compression based on functional unit sensitivity, modality-aware decoding with relaxed verification, progressive state management for streaming continuity, and stage-disaggregated serving through hardware-algorithm co-design. Our literature repository is at https://github.com/SuDIS-ZJU/Efficient-LVLMs-Inference.

## 一、研究背景与核心问题

- **研究动机**：  
  大规模视觉语言模型（LVLMs）在图像与视频理解、跨模态推理方面获得显著成功，但推理成本极高。主要瓶颈源于所谓的“视觉标记主导（visual token dominance）”现象——视觉输入（上千标记）的规模远超文本输入（仅几十标记）。
- **核心问题**：  
  与纯文本大模型不同，LVLM 推理受到计算密度、注意力平方复杂度与显存带宽的多重约束，导致多阶段性能瓶颈：
  1. **编码阶段（Encoding）**：由高分辨率视觉特征提取引起的计算密集负载。  
  2. **预填阶段（Prefilling）**：由长上下文注意力导致的计算与显存写入双重耗时。  
  3. **解码阶段（Decoding）**：因静态视觉 Key-Value 缓存（KV Cache）巨大而形成“视觉记忆墙”（Visual Memory Wall）。  
- **研究目标**：  
  构建一个**阶段感知（stage-aware）**的系统化框架，对端到端 LVLM 推理效率进行分析与分类，指导算法优化与系统设计。

## 二、方法论：三阶段推理分析与系统化技术框架

### 1. 整体框架
论文提出基于推理生命周期的三阶段建模：

\[
\tau_{total} = \tau_{ENC} + \tau_{PFL} + N_o \cdot \tau_{DEC}
\]

其中：
- \( \tau_{ENC} \)：视觉编码延迟（决定计算饱和）。
- \( \tau_{PFL} \)：上下文预填充延迟（决定第一 token 响应时间，TTFT）。
- \( \tau_{DEC} \)：单 token 解码延迟（决定每输出 token 延时，TPOT）。

引入**Roofline 模型分析**，将不同阶段映射至算力带宽界限，判定其为：
- 编码阶段：计算受限 (Compute-Bound)；
- 预填阶段：计算与显存双受限；
- 解码阶段：带宽受限 (Memory-Bound)。

### 2. 三阶段优化技术
论文建立了图示化的技术分类（见 Figure 3），归纳如下：

#### （1）编码阶段优化
- **架构优化**：采用轻量或层次视觉编码器（FastViT、ConvLLaVA、FastVLM），以及高效模态适配器（BLIP-2、Qwen-VL 等）以降低初始视觉标记数 \(N_v\)。  
- **输入降维**：关键帧选择、动态分辨率调整（Q-Frame、VisionThink）及编码端标记压缩（如 VisionZip、VisPruner）。  
  → 目标：在保留语义密度的同时削减视觉标记产生量。  

#### （2）预填阶段优化
- **标记压缩**：在 LLM 投影空间中进行跨模态标记合并。分为：
  - 多样性驱动（G-Prune、DyCoke、DivPrune）：删除冗余视觉区域；
  - 注意力驱动（FastV、SparseVLM、ATP-LLaVA）：依据注意力权重裁剪。  
  - 趋势：二者结合的混合压缩。  
- **稀疏注意力**：通过构建稀疏模式（XAttention、VideoNSA）降低注意力计算复杂度，从 \(O(n^2)\) 降至块级或分层复杂度。  

#### （3）解码阶段优化
- **KV 缓存压缩**：在不同粒度进行：
  - token级（LOOK-M、ReKV）、层级（VL-Cache、MEDA）、头级（SparseMM）、比特级（VidKV、AKVQ-VL）；  
  - 利用模态感知压缩策略，对视觉缓存进行激进压缩，而保持文本缓存高精度。  
- **推测解码（Speculative Decoding）**：通过“草稿 + 验证”机制并行生成，提高吞吐。视觉场景中进一步采用视觉轻量草稿模型（Spec-LLaVA、SpecVLM、FASTVLM）和宽松验证机制。  
- **高效推理**：减少输出长度（如 PixelThink、CAR）以提升整体推理速度。

## 三、实验设计与评估体系

### 1. 实验场景与数据集
- 使用 Qwen2.5-VL、Qwen3.5、VideoChatGPT 等多模态模型作为主干；
- 基准测试涵盖 **图像与视频两类任务**：
  - 图像类：SlideVQA、MMCoQA、WebQA、Multi-ModalQA；
  - 视频类：VideoChatGPT、VideoDetailCaption、LongVideoBench 等。
- 系统层面评估包括 TTFT（首 token 时延）、TPOT（单 token 持续时延）、Goodput（吞吐率）等指标。

### 2. 对比基线
- 编码优化对比传统模型（如 CLIP、LLaVA）；
- KV 缓存压缩对比三类现有方法：LOOK-M、MadaKV、SparseMM；
- 推测解码实验对比自回归基线和 Visual Token Pruning 前后性能。  

### 3. 实验内容
- **混合 KV 缓存压缩实验**：验证自研“Hybrid KV”方案，对比中在相同条件下减少到 20% 缓存预算时，推理延迟下降约 36%，准确率保持 >97%。  
- **宽松推测解码实验**：在 Video Detail Caption 基准上实现 1.8–2.6× 加速，同时保留 81–97% 输出质量。  
- 实验覆盖多尺度场景：静态图像、长视频、流式视频。

## 六、主要结论与发现

1. **LVLM 推理存在跨阶段瓶颈转移**：优化单一环节常导致其他阶段的次生瓶颈。  
2. **编码阶段决定整体效率基线**：早期 token 压缩能带来最大端到端收益。  
3. **预填阶段的平方复杂度是响应延迟的核心来源**。  
4. **解码受制于视觉记忆墙**，而非算力；视觉 KV 占总缓存 80% 以上。  
5. **下一代优化方向**：
   - 混合压缩（多策略组合），  
   - 模态感知解码与宽松验证，  
   - 流式渐进状态管理，  
   - 硬件-算法协同的阶段解耦式服务架构。
