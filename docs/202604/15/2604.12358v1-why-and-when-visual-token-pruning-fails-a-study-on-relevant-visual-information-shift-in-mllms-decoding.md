## 摘要
近年来，视觉标记剪枝（visual token pruning）被提出以应对多模态大语言模型（MLLMs）中大量视觉标记的问题。然而，我们观察到，现有的剪枝方法在简单的视觉理解任务中表现可靠，但在复杂视觉推理任务上难以有效泛化，这一关键问题在先前研究中尚未得到充分探讨。通过系统分析，我们发现了解码阶段的相关视觉信息偏移（Relevant Visual Information Shift, RVIS）是导致剪枝失败的主要原因。为此，我们提出了一种无需训练的附加框架——解码阶段感知偏移的标记剪枝方法（Decoding-stage Shift-aware Token Pruning, DSTP），能够在解码过程中使现有的剪枝方法动态地将视觉标记与不断变化的推理需求对齐。大量实验证明，DSTP 能够显著减轻剪枝方法在复杂推理任务中的性能退化，同时在视觉理解基准测试中也持续带来性能提升。此外，DSTP 在多种最新架构上均表现出有效性，突显了其良好的泛化能力和高效性，同时计算开销极小。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

## 一、研究问题与背景

- **研究动机**：多模态大语言模型（MLLMs）需要处理大量由视觉编码器产生的视觉 token，这造成了推理时的计算与内存开销。为此，学界提出了视觉标记剪枝（Visual Token Pruning）方法，以减少冗余视觉信息、提升推理效率。  
- **问题发现**：现有剪枝方法在简单的视觉理解任务（如 VQA）中表现良好，但在复杂视觉推理任务（如视觉数学推理）中性能急剧下降。  
- **核心问题**：通过分析模型的视觉注意力变化，作者发现解码阶段存在视觉注意力的动态迁移现象——**相关视觉信息偏移（Relevant Visual Information Shift, RVIS）**。这意味着模型在推理过程中不断改变其关注图像区域，而现有剪枝方法仅在预填阶段（prefill stage）静态选择 token，未能适应这种动态变化，从而导致性能下降。

## 二、方法论：DSTP 框架

### 1. 核心思想
- 提出一种**无训练、解码阶段感知偏移的标记剪枝框架（Decoding-stage Shift-aware Token Pruning, DSTP）**。
- DSTP 旨在动态检测并适应 RVIS，从而在推理过程中恢复或替换被剪枝但重新变得有关的视觉 token。

### 2. 模块组成
DSTP 包含两个关键模块：

#### （1）RVIS 检测模块：Relevant Visual Information Shift Detect（RISD）
- **功能**：在解码阶段监控视觉注意力分布的相似度，判断视觉注意力是否与初始（prefill 阶段）关注区域发生显著偏离。
- **计算方式**：基于余弦相似度  
  \[
  s_l = \frac{a_{t_0 \to v} \cdot a_{t_l \to v}}{|a_{t_0 \to v}| |a_{t_l \to v}|}
  \]
  当相似度 \( s_l < \tau \)（阈值）时，判定为出现 RVIS。

#### （2）上下文保持视觉标记替换模块：Context-Preserving Visual Token Swap（CPTS）
- **功能**：在检测到 RVIS 时，重新评估所有视觉 token 的重要性，找出当前解码步骤需要的新 token。
- **机制**：
  - 保留被剪枝的 token 作为候选集合；
  - 当检测到新的需求时，通过合并操作  
    \[
    X_{\text{CP},l}^v = X_{\text{Pruned}}^v \cup X_{\text{Pruned},l}^v
    \]
    形成上下文保持的视觉 token 集（而非完全替换），避免上下文丢失；
  - 在持续 L 步解码后重新回到初始剪枝集合，以节约算力。

### 3. 算法流程（文字描述）
1. **Prefill 阶段**：利用基础剪枝算法（如 FastV、DivPrune）选出 top-k 重要视觉 token，形成剪枝集合并保留其余 token 作为备用。  
2. **Decoding 阶段**：逐步生成文本，每一步计算当前视觉注意力与初始注意力的相似度。  
3. 当检测到相似度下降（RVIS 出现）时：
   - 启动 CPTS：重新评估全部视觉 token；
   - 合并新选择 token，与原 token 保持上下文连续；
   - 维持合并状态 L 步后回退到初始剪枝集合。
4. **输出推理文本结果**。

### 4. 特点
- 完全无训练；
- 轻量化，计算代价极低；
- 可即插即用，与现有剪枝方法、任意 MLLM 主干兼容。

## 六、主要结论与发现

1. 解码阶段存在 **相关视觉信息偏移（RVIS）**，是剪枝失败的根本原因。  
2. RVIS 在复杂推理任务中频繁出现，而在简单视觉理解中几乎没有出现。  
3. 静态剪枝方法无法动态响应这种视觉焦点变化，导致性能骤降。  
4. 所提出的 **DSTP 框架** 能实时检测注意力偏移并恢复关键视觉 token，使得模型在复杂推理任务上性能提升 30～40%，在视觉理解任务上亦有稳定增益。  
5. DSTP 的效率开销极低，保持剪枝加速优势的同时显著提升准确率。  
6. 该框架展现出良好的模型兼容性和任务泛化性。

## 八、不足与局限

- **层级统一问题**：DSTP 在所有 Transformer 层使用相同 token 集，未区分浅层（低级视觉特征）与深层（语义推理）之间的差异。  
- **静态超参数**：检测阈值（τ）与上下文保留长度（L）为固定值，未实现动态自适应。  
- **实时决策能力有限**：当前仅通过余弦相似度判断偏移，有待探索更复杂或学习型策略（如强化学习触发机制）。  
- **应用场景**：目前主要验证在图像问答与视觉数学任务上，尚未涵盖视频或跨时序场景。  
- **解释挑战**：虽然可视化支持了理论解释，但推理过程中的注意力动态仍存在非确定性因素。
