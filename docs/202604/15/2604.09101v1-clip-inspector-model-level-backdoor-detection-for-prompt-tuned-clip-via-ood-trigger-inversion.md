## 摘要
在数据和计算资源有限的情况下，组织越来越多地将模型训练外包给“机器学习即服务”（MLaaS）提供商。这类提供商通常通过提示微调（prompt tuning）而非从头训练的方式，将视觉-语言模型（VLMs）如 CLIP 适配到下游任务。然而，这种“半诚实”场景带来了安全风险：恶意提供商虽遵循提示微调协议，却可能植入后门，使得带有特定触发器的输入被强制分类为攻击者指定的类别，即便是分布外（OOD）的数据，也会受影响。此类后门保持编码器不变，因此现有专注于编码器篡改的检测方法无法发现；而那些在训练前或推理时清理数据的“数据级”方法，也无法回答关键问题——“交付的模型是否已被植入后门？” 为解决这一模型级验证问题，我们提出了 CLIP-Inspector（CI），一种专为提示微调 CLIP 模型设计的后门检测方法。假设我们对白盒访问交付模型并拥有一组未标注的 OOD 图像，CI 通过为每个类别重建潜在触发器，判断模型是否表现出后门行为。此外，我们还展示了，利用 CI 重建的触发器对带有正确标签的触发输入进行微调，可重新校准模型并削弱后门效果。通过在十个数据集和四种后门攻击上的大量实验，我们证明 CI 仅利用 1000 张 OOD 图像即可在一个 epoch 内重建有效触发器，并实现 94% 的检测准确率（47/50 模型）。与改进的触发反演基线相比，CI 的 AUROC 得分显著更高（0.973 对比 0.495/0.687），从而使提示微调 CLIP 模型的审查与事后修复成为可能，确保其安全部署。

## Abstract
Organisations with limited data and computational resources increasingly outsource model training to Machine Learning as a Service (MLaaS) providers, who adapt vision-language models (VLMs) such as CLIP to downstream tasks via prompt tuning rather than training from scratch. This semi-honest setting creates a security risk where a malicious provider can follow the prompt-tuning protocol yet implant a backdoor, forcing triggered inputs to be classified into an attacker-chosen class, even for out-of-distribution (OOD) data. Such backdoors leave encoders untouched, making them undetectable to existing methods that focus on encoder corruption. Other data-level methods that sanitize data before training or during inference, also fail to answer the critical question, "Is the delivered model backdoored or not?" To address this model-level verification problem, we introduce CLIP-Inspector (CI), a backdoor detection method designed for prompt-tuned CLIP models. Assuming white-box access to the delivered model and a pool of unlabeled OOD images, CI reconstructs possible triggers for each class to determine if the model exhibits backdoor behaviour or not. Additionally, we demonstrate that using CI's reconstructed trigger for fine-tuning on correctly labeled triggered inputs enables us to re-align the model and reduce backdoor effectiveness. Through extensive experiments across ten datasets and four backdoor attacks, we demonstrate that CI can reconstruct effective triggers in a single epoch using only 1,000 OOD images, achieving a 94% detection accuracy (47/50 models). Compared to adapted trigger-inversion baselines, CI yields a markedly higher AUROC score (0.973 vs 0.495/0.687), thus enabling the vetting and post-hoc repair of prompt-tuned CLIP models to ensure safe deployment.

  随着视觉-语言模型（Vision-Language Models, VLMs）如 CLIP 在下游任务中的广泛使用，越来越多的机构选择依托机器学习即服务（MLaaS）平台进行提示微调（Prompt Tuning）。然而，这种“半诚实”外包训练机制存在重大安全风险：服务商可能在遵循规范的表面下，将后门（Backdoor）隐蔽地植入模型，使得携带某种特定“触发信号”（Trigger）的输入被错误地分类为攻击者选择的类目，即使输入属于分布外数据（OOD）也会触发此行为。

- **问题关键点**：
  - 提示微调的 CLIP 模型通常只调整小型元网络（meta-net），保持主干编码器冻结。这意味着传统依赖检测编码器篡改的后门检测方法失效。
  - 数据清洗类（Data-level）防御或推理阶段防御方法无法回答关键问题：“交付的模型是否被后门污染？”
  - 需要开发**面向模型级别**、能在部署前验证 CLIP 是否被污染的检测工具。

- **研究目标**：  
  提出一种能够在 **白盒访问条件下、使用少量未标注 OOD 图片**，检测并识别提示微调 CLIP 模型是否被嵌入后门的可靠方法。

## 二、核心方法 —— CLIP-Inspector (CI)

### 1. 方法总体思路

- CI 专为提示微调 CLIP 场景设计，旨在：
  1. 检测模型中是否存在“后门行为”；
  2. 推断被攻击的目标类别；
  3. 在检测后，可利用重建的触发器执行模型“恢复”（再微调以削弱后门）。

- **关键假设**：
  - 拥有模型权重和架构（白盒访问）。
  - 无需任何训练或标注数据，仅依赖少量 OOD 未标注图像（约 1000 张）。

### 2. 技术细节

- **触发器反演（Trigger Inversion）**：  
  对于每个候选类别 \( c \)，CI 通过梯度下降优化一个全图像触发扰动 \(\delta_c\)，其约束为 \( \|\delta_c\|_\infty \le 4/255 \) 保证不可察觉。优化目标：
  \[
  \min_\delta L_{margin}(\delta) = - \frac{1}{|D|} \sum_{x\in D}\left[l_c(x+\delta) - \max_{c' \ne c} l_{c'}(x+\delta)\right]
  \]
  其中 \(l_c(x)\) 是 CLIP 模型中该类的 logit，相当于**提升目标类 logit 与最大竞争类 logit 的差距**。  
  → 优化完成后，计算触发样本的攻击成功率（ASR）。

- **异常分数计算（Anomaly Scoring）**：
  - 每类计算两个指标：
    1. **ASR（Attack Success Rate）**：越高表示更容易触发后门。
    2. **平均优化损失（Average Loss）**：损失越小表示模型很容易“被引导”。
  - 综合得分：
    \[
    S(c) = z_{\text{ASR}}(c) - z_{\text{loss}}(c)
    \]
    再标准化成 z-score；若最大得分 \(s_{\max}\) ≥ 2（默认阈值），判定为后门模型，同时识别对应目标类别。

- **算法流程精简描述**：
  1. 输入模型与未标注 OOD 集；
  2. 对每个候选类别优化一个触发器；
  3. 评估其 ASR 与损失；
  4. 汇总异常分数；
  5. 若存在明显异常类，则判为 backdoor。

- **后门修复功能**：
  使用检测出的触发器 \(\delta_{\hat{c}}\)，在存在少量干净标注样本的条件下，对 (x + δ̂c, y) 进行再训练，可有效对齐模型决策边界，显著削弱后门效果而不损失准确度。

## 三、实验设计

### 1. 数据集与任务设定

- 共测试 **10 个数据集**：
  ImageNet, Caltech101, OxfordPets, Flowers102, Food101, FGVC-Aircraft, EuroSAT, SUN397, DTD, UCF101。  
  均为标准图像分类任务。

- **模型架构**：
  - CLIP ViT-B/16 作为基础模型；
  - 采用 CoCoOp（Conditional Context Optimization）进行提示微调；
  - 保持图像与文本编码器冻结，仅更新小型 MLP 元网络。

- **测试协议**：
  - Seen / Unseen 分类：在 Seen 类微调，在 Unseen 类测试，以模拟开放集部署。
  - 每类仅有 16 张样本（few-shot）。

- **OOD 图像来源**：Open Images（无类别重叠）。

### 2. 攻击设置

- 覆盖 **4 种典型后门攻击**：
  1. **BadCLIP**（专为 CoCoOp 提示微调攻击设计）；
  2. **Blended**（图像加噪混合触发）；
  3. **SIBA**（稀疏隐形触发器）；
  4. **WaNet**（几何形变式触发）。

- 每次攻击污染 10% 训练图像，触发范数约束 \( \ell_\infty = 4/255 \)。

### 3. 对比方法（Baselines）

- **Neural Cleanse (NC)** [Wang et al. 2019]：基于掩码稀疏假设。
- **Pixel Backdoor (PixB)** [Tao et al. 2022]：基于像素级稀疏触发优化。  
  （均为目前常用的触发反演类模型级检测器，但假设不适用于 CLIP）

### 4. 度量指标

- 模型级指标：**AUROC** （清洁 vs 后门模型的区分能力）。
- 类别级指标：**F1-score**（是否正确识别目标类别）。
- 运行效率评估（检测时长）。
- 在修复实验中评估 ACC（准确率）与 ASR（攻击成功率）。

## 六、主要发现与结论

1. **检测性能卓越**：
   - CI 在 50 个模型中检测准确率达 **94%**；
   - 平均 AUROC = **0.973**（对比基线 NC=0.495, PixB=0.687）；
   - 类别识别 F1 = 0.92（几乎无误报）。

2. **泛化能力强**：
   - 即使后门嵌入编码器层（非提示网络），CI 仍然能检测；
   - 自适应攻击（BadCLIP Adaptive）下仍可检测大多数模型。

3. **修复能力显著**：
   - 使用重建触发器再微调后，平均 ASR 从 84% 降至 **6% 以下**；
   - Clean accuracy 几乎不降（±1%）。

4. **效率与可操作性**：
   - 单 epoch 即收敛；
   - 无需训练数据和标签即可做检测。
