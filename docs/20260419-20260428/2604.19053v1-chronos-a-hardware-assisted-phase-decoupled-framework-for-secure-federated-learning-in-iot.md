## 摘要
我们提出了 CHRONOS，这是一种硬件辅助框架，用于将私有梯度聚合所需的加密设置从主动训练阶段中解耦。CHRONOS 在设备空闲窗口期间通过服务器中继执行每个 epoch 一次的 Diffie-Hellman 密钥交换。它在 ARM TrustZone 隔离环境中生成临时密钥对并派生伪随机生成(PRG)密钥，确保私钥永不出现在普通世界内存中。成对秘密在隔离环境中被封装，临时私钥的 Shamir 秘密份额被分发给其他节点。在训练阶段，客户端通过单次流密码评估掩盖梯度，并在一次通信回合中传输。由硬件支持的回合计数器保证单次使用的新鲜性。如果客户端在回合中途掉线，服务器可通过存储在其他节点的 Shamir 份额重建其掩码，从而在无需重复该回合的情况下保持聚合正确性。在基于 OP-TEE 的 Rock Pi 4 设备上进行的评估表明，CHRONOS 能够抵御操作系统级的入侵，并有效防御最新的梯度反演攻击。与同步安全聚合相比，对于 20 个客户端，CHRONOS 将主动阶段的聚合延迟减少最多达 74%。该系统在安全世界中保持的永久存储占用低于每设备 700 字节，并且其可扩展性与模型维度无关。

## Abstract
We propose CHRONOS, a hardware-assisted framework that decouples the cryptographic setup required for private gradient aggregation from the active training phase. CHRONOS executes a once-per-epoch server-relayed Diffie-Hellman key exchange during a device's idle window. It generates ephemeral keypairs and derives PRG keys entirely within an ARM TrustZone enclave, ensuring private keys never exist in Normal World memory. Pairwise secrets are sealed in the enclave, and Shamir secret shares of the ephemeral private key are distributed to peers. During training, clients mask gradients with a single stream-cipher evaluation and transmit them in one communication round. A hardware-backed round counter enforces single-use freshness. If clients drop out mid-round, the server reconstructs their masks from peer-held Shamir shares, preserving correct aggregation without repeating the round.   Evaluation on Rock Pi 4 devices using OP-TEE demonstrates that CHRONOS achieves OS-level compromise resistance and thwarts state-of-the-art gradient inversion attacks. It reduces active-phase aggregation latency by up to 74% compared to synchronous secure aggregation for 20 clients. The system maintains a persistent Secure World storage footprint of fewer than 700 bytes per device, scaling independently of model dimension.

## 一、研究背景与核心问题

- **研究动机**  
  在物联网（IoT）场景中，设备数量庞大且资源有限，许多终端负责采集高敏感数据（如健康监测、环境传感器等）。  
  联邦学习（Federated Learning, FL）允许这些设备在不上传原始数据的前提下协同训练模型，然而客户端上传的**梯度更新**仍然会泄露隐私。  
  已有研究指出，攻击者（尤其是“诚实但好奇”服务器）可以通过**梯度反演攻击（Gradient Inversion Attack）**重构原始训练样本。  

- **问题与挑战**  
  传统防御方法（如 Secure Multiparty Computation, SMPC 或 Secure Aggregation, SecAgg）虽然保护隐私，但**通信与计算开销巨大**，尤其是在每轮训练都需要多节点交互的情况下。  
  对于资源受限的 IoT 节点，这种高成本同步协议**无法满足实时性和能效要求**。  

- **研究目标**  
  论文提出框架 **CHRONOS**，通过**硬件辅助的阶段解耦（Phase-Decoupling）机制**：
  - 将安全聚合的加密密钥协商移出训练阶段；
  - 结合 ARM TrustZone 形成硬件可信环境（TEE），实现 OS 抵抗级别的安全；
  - 兼顾**低延迟、低能耗与高隐私保护**。

## 二、方法概述与技术细节

### 1. 核心思想

> 在设备“空闲期”提前完成密钥协商，在“训练期”仅执行一次轻量加密传输。

### 2. 系统架构

CHRONOS 将每个 IoT 客户端分为两部分：
- **Normal World（普通环境）**：运行系统与训练逻辑；
- **Secure World（安全环境）**：Trusted Application (TA)，基于 ARM TrustZone 实现，负责所有密钥生成、封装与计数器更新。

### 3. 双阶段操作流程

#### （1）Idle Phase（空闲阶段）
- 每个设备在空闲时于 TEE 内生成一次**临时椭圆曲线密钥对**；
- 通过服务器中继的 **Diffie-Hellman (DH)** 协议与其他客户端建立成对共享密钥；
- 使用 **HKDF** 派生两个类型的密钥：
  - 加密密钥 \( k_{enc} \)：用于加密 Shamir 秘密份额；
  - 伪随机生成密钥 \( k_{prg} \)：用于梯度掩码。
- 将所有共享密钥封装（seal）进 TEE 安全存储，仅保存所需 632 字节；
- 通过 **Shamir Secret Sharing** 分发私钥份额，用于潜在的客户端掉线恢复。

#### （2）Active Phase（训练阶段）
- 每客户端计算本地梯度 \( g_i \)，调用 TA 生成掩码：
  \[
  m_i(r) = \sum_{j > i} PRG(s_{ij}, r) - \sum_{j < i} PRG(s_{ji}, r)
  \]
  其中 \( PRG \) 为基于 AES-128-CTR 的伪随机生成器。
- 用掩码加密梯度 \( \tilde g_i = g_i + m_i(r) \) 并一次性发送给服务器。
- 服务器直接求和得到正确的加权聚合结果：
  \[
  \sum_i \tilde g_i = \sum_i g_i + \sum_i m_i = \sum_i g_i
  \]
- 若有客户端掉线，服务器通过收集 \( t \) 个 Shamir 份额恢复其掩码继续聚合，无需重启训练轮。

### 4. 安全机制

- **OS 级防御**：私钥生成、存储与掩码生成全在 TEE 内，普通系统即使被攻破也无法提取密钥。  
- **防重放机制**：硬件单调计数器（RPMB 计数器）确保掩码不复用。  
- **Dropout 恢复**：允许最多 \( N-t \) 客户端中途掉线而不丢失聚合正确性。  

### 5. 安全与性能特性
- 单轮加密通信（无需多轮同步）；  
- 安全存储常驻内存 < 700B，与模型维度 \( D \) 无关；  
- 每轮只需一次 TA 调用（<150 ms）。

## 六、主要结论与发现

- 在 20 客户端场景下，CHRONOS **比 SecAgg 减少 74% 延迟**；
- 活动阶段能量消耗减少 3.8 倍；
- 在 TrustZone 内驻留数据仅 632 字节；
- 可容忍多达 **N−t 客户端掉线**，恢复通信开销极低；
- 对抗梯度反演攻击时，重建图像降至**PSNR ≈ 8 dB（近似随机噪声）**；
- 在 OS 被攻破的前提下仍保持梯度隐私。

## 七、主要优点

- **硬件隔离**：保障对恶意操作系统的强隐私隔离；
- **相位解耦**：密钥预计算显著降低同步通信开销；
- **极小 TEE 占用**：<700B 永久存储，无模型维度依赖；
- **快速恢复机制**：Shamir 恢复仅需一次额外交互；
- **可扩展性优越**：通信复杂度由 O(N²) 降至 O(N)。
