## 摘要
联邦学习（Federated Learning, FL）允许多个客户端在不共享其私有训练数据的情况下，通过服务器协作训练一个全局机器学习模型。在传统的联邦学习中，系统采用同步方式，服务器在聚合众多客户端的模型更新以更新全局模型之前，需要等待所有客户端的模型提交。然而，同步联邦学习受到慢节点（straggler）问题的制约。为了解决这一问题，异步联邦学习（Asynchronous FL）架构允许服务器在接收到任意客户端的本地模型更新后立即更新全局模型。尽管这种方法具有优势，但异步联邦学习的去中心化特性使其易受投毒攻击。针对异步联邦学习已提出多种防御机制，但这些方法仍容易受到高级攻击或依赖于不切实际的服务器假设。本文提出了SecureAFL，一个专为抵御投毒攻击而设计的创新异步联邦学习安全框架。SecureAFL通过检测并丢弃异常更新，同时估计缺失客户端的贡献，从而提升异步联邦学习的鲁棒性。此外，它采用了如坐标中值（coordinate-wise median）等拜占庭鲁棒聚合技术来整合已接收和估计的更新。大量基于真实数据集的实验结果表明，SecureAFL在提升防御能力方面效果显著。

## Abstract
Federated learning (FL) enables multiple clients to collaboratively train a global machine learning model via a server without sharing their private training data. In traditional FL, the system follows a synchronous approach, where the server waits for model updates from numerous clients before aggregating them to update the global model. However, synchronous FL is hindered by the straggler problem. To address this, the asynchronous FL architecture allows the server to update the global model immediately upon receiving any client's local model update. Despite its advantages, the decentralized nature of asynchronous FL makes it vulnerable to poisoning attacks. Several defenses tailored for asynchronous FL have been proposed, but these mechanisms remain susceptible to advanced attacks or rely on unrealistic server assumptions. In this paper, we introduce SecureAFL, an innovative framework designed to secure asynchronous FL against poisoning attacks. SecureAFL improves the robustness of asynchronous FL by detecting and discarding anomalous updates while estimating the contributions of missing clients. Additionally, it utilizes Byzantine-robust aggregation techniques, such as coordinate-wise median, to integrate the received and estimated updates. Extensive experiments on various real-world datasets demonstrate the effectiveness of SecureAFL.

## 一、核心问题与研究背景

- **研究动机**：  
  联邦学习（Federated Learning, FL）允许多个客户端在保证数据不出本地的前提下，共同训练全局模型。传统 FL 通常采用 **同步机制**，即服务器等待所有或大部分客户端上传模型更新后再进行聚合。然而，该机制容易受到 **straggler（慢节点）问题** 的影响，导致训练延迟、效率低下。  
  异步联邦学习（Asynchronous FL, AFL）允许服务器在接收到任意客户端的更新后立即更新全局模型，从而提高训练效率。但异步机制带来的 **更新延迟与模型版本不一致** 使系统更容易遭受 **中毒攻击（poisoning attacks）**。

- **研究问题**：  
  异步 FL 缺乏可同时应对延迟与恶意更新的鲁棒防御机制。现有防御方法要么假设服务器拥有可信数据集（不现实），要么在面对强对抗性攻击时仍脆弱。

- **研究目标**：  
  提出一种通用且有效的安全框架 **SecureAFL**，在完全异步的训练环境下增强系统对恶意客户端中毒攻击的防御能力，同时保持模型收敛性与精度。

## 二、方法论与关键技术

### 1. 核心思想
SecureAFL 从三个层面强化异步联邦学习的安全性：

1. **基于平滑性约束的更新过滤（Filtering）**：  
   利用局部模型更新的 Lipschitz 连续性来检测异常更新。若一个客户端的更新变化过于剧烈且超出历史平滑范围，则认为其可能被污染并丢弃。

2. **缺失客户端更新估计（Estimation）**：  
   对未上传更新的客户端，服务器根据历史的全局模型变化与客户端的过去更新趋势，使用近似二阶信息（通过 L-BFGS 方法）估计这些缺失更新，从而保持聚合的完整性。

3. **拜占庭鲁棒聚合（Robust Aggregation）**：  
   在整合真实与估计更新时，采用 **坐标中值（coordinate-wise median）** 方式，以减轻极端值或恶意更新对全局模型的影响。

### 2. 算法流程概要（服务器端伪流程）
- 初始化全局模型并分发给各客户端。
- 对每轮更新：
  1. 接收某客户端上传的局部更新；
  2. 计算其 Lipschitz 因子并与历史分布比较；
  3. 若该更新平滑（满足百分位阈值），则与其他客户端估计更新一并聚合；
  4. 若被判为异常，则仅使用估计更新进行聚合；
  5. 最后更新全局模型并发送给客户端。  
- 特殊初始化阶段使用 ℓ₂ 范数剪裁，防止过大梯度影响。

### 3. 理论保证
论文提供了 **非凸优化条件下的收敛性分析**，证明在异步环境和拜占庭客户端存在的情况下，只要步骤满足一定条件（学习率 η ≤ 1/(4L)），则平均梯度范数可以收敛到一个有界区间，误差项与估计误差、噪声和延迟上界成正比。

## 八、不足与局限

- **隐私隐忧**：服务器通过历史更新估计客户端行为，可能推断局部目标函数变化，存在推理性隐私风险；
- **计算与存储开销**：需保存历史缓冲区（规模 O(nεd)），增加一定服务器负载；
- **假设条件较强**：收敛性分析依赖“多数客户端为善意”（n > 2b + 1），实际部署可能不易保证；
- **实验环境限制**：仅在单机 GPU 上验证，缺少分布式大规模真实系统测试；
- **Adaptive 攻击定义依赖现有防御的先验知识，实际复杂度可能更高**；
- **隐含参数如 α、G、ε 需人工搜索，真实部署可能增加调参成本。**
