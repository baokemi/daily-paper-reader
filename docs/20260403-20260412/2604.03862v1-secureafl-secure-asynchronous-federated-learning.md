## 摘要
联邦学习（Federated Learning，FL）使多个客户端能够通过服务器协作训练一个全局机器学习模型，而无需共享各自的私有训练数据。在传统的 FL 系统中，采用同步方式，服务器需等待众多客户端的模型更新后再进行聚合以更新全局模型。然而，同步 FL 常常受到“慢客户端”问题的阻碍。为了解决这一问题，异步 FL 架构允许服务器在接收到任意客户端的本地模型更新后立即更新全局模型。尽管异步 FL 具有优势，但其去中心化特性使其易受到中毒攻击。虽然已有针对异步 FL 的多种防御机制被提出，但这些方法仍易受高级攻击影响，或依赖于不切实际的服务器假设。本文提出 SecureAFL，这是一种旨在防御中毒攻击的创新异步 FL 安全框架。SecureAFL 通过检测并舍弃异常更新，同时估计缺失客户端的贡献，提高了异步 FL 的鲁棒性。此外，该框架利用拜占庭鲁棒聚合技术（如逐坐标中位数）整合接收到的更新与估计更新。基于多个真实数据集的广泛实验结果表明，SecureAFL 具有显著的有效性。

## Abstract
Federated learning (FL) enables multiple clients to collaboratively train a global machine learning model via a server without sharing their private training data. In traditional FL, the system follows a synchronous approach, where the server waits for model updates from numerous clients before aggregating them to update the global model. However, synchronous FL is hindered by the straggler problem. To address this, the asynchronous FL architecture allows the server to update the global model immediately upon receiving any client's local model update. Despite its advantages, the decentralized nature of asynchronous FL makes it vulnerable to poisoning attacks. Several defenses tailored for asynchronous FL have been proposed, but these mechanisms remain susceptible to advanced attacks or rely on unrealistic server assumptions. In this paper, we introduce SecureAFL, an innovative framework designed to secure asynchronous FL against poisoning attacks. SecureAFL improves the robustness of asynchronous FL by detecting and discarding anomalous updates while estimating the contributions of missing clients. Additionally, it utilizes Byzantine-robust aggregation techniques, such as coordinate-wise median, to integrate the received and estimated updates. Extensive experiments on various real-world datasets demonstrate the effectiveness of SecureAFL.

## 一、研究背景与核心问题

- **研究动机**：  
  联邦学习（Federated Learning, FL）允许多个客户端在不共享本地数据的前提下，协同训练全局模型。然而，传统 **同步式 FL** 需要等待所有客户端返回更新才进行聚合，容易受到“*慢客户端（straggler）问题*”影响，导致训练效率显著下降。  

- **异步联邦学习（Asynchronous FL, AFL）** 的优势：  
  AFL 不需要等待所有客户端提交更新，服务器在接收到任意客户端的更新后即可更新全局模型，从而提升了训练效率与可扩展性。  

- **核心挑战**：  
  AFL 的异步与去中心化特性导致其**更易受到中毒攻击（poisoning attacks）**，包括：
  - 未及时收到更新的客户端造成模型滞后；
  - 攻击者可通过操纵上传模型更新来毒化全局模型；
  - 异步环境下传统同步防御（如 Krum、Trimmed Mean 等）无法直接应用。  

- **论文目标**：  
  提出一种面向异步联邦学习的安全防御框架——**SecureAFL**，能够在不中断异步机制的前提下：
  - 有效识别异常或恶意更新；
  - 准确估计缺失客户端的更新；
  - 并通过鲁棒聚合提升系统整体安全与稳定性。

## 二、方法论与技术细节

### 1. 核心思想

SecureAFL 综合了 **异常检测、历史估计与拜占庭鲁棒聚合（Byzantine-robust aggregation）** 三个核心模块，实现对异步 FL 的防御增强。方法流程如下：

1. **更新过滤（Filtering）**：  
   基于“**Lipschitz 连续性约束**”，检测更新变化的平滑程度。  
   - 对于每个客户端 \( i \) 在训练轮次 \( t \) 发送的更新 \( g_i^{t-\tau_i} \)，计算历史与当前更新的“平滑系数”：
     \[
     \lambda_i^t = \frac{ \| g_i^{t-\tau_i} - g_i^\varphi \| }{ \| w_{t-\tau_i} - w_\varphi \| }
     \]
   - 若 \( \lambda_i^t \) 超出历史分布的 α 百分位阈值 \( Q_t^\alpha \)，则认为该更新为异常并予以丢弃。

2. **缺失更新估计（Estimation）**：  
   当部分客户端未及时上传更新时，服务器使用近似二阶信息（Hessian 估计）来**重建其缺失更新**。  
   采用 **L-BFGS（Limited-memory BFGS）** 方法近似局部 Hessian，从历史的模型参数变化 Δw 与更新差 Δg 估计出：
   \[
   \hat{g}_k^t = g_k^v + H_k^t (w_t - w_v)
   \]
   从而推断当前迭代下客户端 k 的可能梯度。

3. **鲁棒聚合（Robust Aggregation）**：  
   使用 **逐坐标中位数（Coordinate-wise Median）** 聚合所有有效的客户端更新，防止极端值或恶意更新影响。  
   - 若更新通过过滤条件：整合真实 + 估计更新；
   - 若被过滤：仅聚合估计更新。

4. **整体算法流程（Algorithm 3）**：
   - 初始轮：进行 L2 范数裁剪以防异常值；
   - 每轮：检查更新 → 判断是否恶意 → 合并真实与估计更新 → 更新全局模型；
   - 迭代直至收敛。

### 2. 理论分析

- 提供了在非凸情形下的**收敛性证明**，给出以下**收敛上界**：
  \[
  \frac{1}{T}\sum_{t=0}^{T-1}E\|\nabla F_H(w_t)\|^2 \leq O(\frac{1}{\sqrt{T}}) + O(E^2_{track})
  \]
  其中 \(E_{track}\) 衡量异步延迟、估计误差与统计噪声的综合影响。保证了在有界延迟与有限恶意客户端（<50%）条件下的收敛稳定性。
