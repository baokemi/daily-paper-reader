## 摘要
联邦学习（Federated Learning, FL）能够在保护数据隐私的同时实现协作式模型训练，但其实际部署受到系统与统计异质性的阻碍。虽然联邦网络剪枝提供了一种缓解该问题的途径，但现有方法面临着关键性两难困境：服务器端剪枝缺乏个性化，而客户端剪枝对于资源受限的设备来说计算代价高昂。此外，剪枝过程本身还会在异构子模型之间产生显著的参数分歧，从而导致训练不稳定并阻碍全局收敛。为应对这些挑战，我们提出了 SubFLOT，这是一种基于服务器端个性化联邦剪枝的新框架。SubFLOT 引入了最优传输增强剪枝（Optimal Transport-enhanced Pruning, OTP）模块，将历史客户端模型视为本地数据分布的代理，将剪枝任务形式化为一个通过最小化 Wasserstein 距离来生成个性化子模型的问题，而无需访问原始数据。同时，为了抵消参数分歧，我们的基于缩放的自适应正则化（Scaling-based Adaptive Regularization, SAR）模块能自适应地惩罚子模型与全局模型的偏离程度，其惩罚强度根据客户端的剪枝率进行缩放。全面的实验结果表明，SubFLOT 持续且显著地优于当前最先进的方法，突显了其在资源受限边缘设备上部署高效且个性化模型的潜力。

## Abstract
Federated Learning (FL) enables collaborative model training while preserving data privacy, but its practical deployment is hampered by system and statistical heterogeneity. While federated network pruning offers a path to mitigate these issues, existing methods face a critical dilemma: server-side pruning lacks personalization, whereas client-side pruning is computationally prohibitive for resource-constrained devices. Furthermore, the pruning process itself induces significant parametric divergence among heterogeneous submodels, destabilizing training and hindering global convergence. To address these challenges, we propose SubFLOT, a novel framework for server-side personalized federated pruning. SubFLOT introduces an Optimal Transport-enhanced Pruning (OTP) module that treats historical client models as proxies for local data distributions, formulating the pruning task as a Wasserstein distance minimization problem to generate customized submodels without accessing raw data. Concurrently, to counteract parametric divergence, our Scaling-based Adaptive Regularization (SAR) module adaptively penalizes a submodel's deviation from the global model, with the penalty's strength scaled by the client's pruning rate. Comprehensive experiments demonstrate that SubFLOT consistently and substantially outperforms state-of-the-art methods, underscoring its potential for deploying efficient and personalized models on resource-constrained edge devices.

  - 服务器端剪枝虽能减小模型，但缺乏对本地数据的个性化适应。

  - 剪枝异构导致**子模型参数分歧**，影响聚合稳定性与全局训练收敛。

- **核心研究问题：**  
  如何在**无需访问本地数据**的情况下，于服务器端生成个性化剪枝模型，同时管理系统与统计异质性，并保持模型收敛稳定性？

## 二、方法论概述

**论文提出 SubFLOT 框架（Submodel Extraction for Efficient and Personalized Federated Learning via Optimal Transport）**，综合利用最优传输（Optimal Transport, OT）与自适应正则化，实现高效个性化的服务器端剪枝。

### 1. 框架总体思路

SubFLOT 包含三个核心模块：

1. **OTP（Optimal Transport-enhanced Pruning）——服务器端个性化剪枝**  
   - 将历史客户端模型视为对应本地数据分布的代理。  
   - 通过最小化全局模型与历史模型间的 **Wasserstein 距离** 来生成个性化剪枝方案。  
   - 采用分层最优传输（layer-wise OT）映射，在神经元级别进行参数对齐，实现差异化子模型生成。

2. **SAR（Scaling-based Adaptive Regularization）——缩放式自适应正则化**  
   - 在客户端训练子模型时引入正则化项：  
     \[
     L_{\text{SAR}}(W_i) = \rho_i \|W_i - \tilde{W}_i\|^2_2
     \]
     其中 \(\rho_i\) 为剪枝率，\(\tilde{W}_i\) 为服务器端生成的初始子模型。  
   - 剪枝率越高，正则项惩罚越强，以抑制过度参数漂移。

3. **OTA（OT-enhanced Aggregation）——最优传输增强聚合**  
   - 在模型聚合阶段，重新计算局部模型与全局模型的 OT 映射，使参数在空间上对齐。  
   - 避免直接加和异构参数造成聚合错误，提高全局模型收敛稳定性。

### 2. 算法流程（文字说明）

1. **服务器端：**
   - 收集上轮客户端模型作为历史模型；
   - 对每个客户端执行 OTP，生成个性化子模型；
   - 将其分发至客户端。

2. **客户端：**
   - 在本地数据上以 SAR 引导的正则化目标训练子模型；
   - 上传更新后的权重。

3. **服务器端聚合：**
   - 通过 OTA 进行子模型参数空间的最优传输对齐；
   - 聚合生成新的全局模型；
   - 重复上述过程直至收敛。
